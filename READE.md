# GD-FramePerfect-Solver (ILL Top-200 Bot)

Автоматизированный математический солвер и макрос-генератор для Geometry Dash (версия 2.2+). Проект разработан для автоматического прохождения сверхсложных уровней из списков ILL (Impossible Levels List) до Топ-200 включительно.

## ⚠️ Дисклеймер / Disclaimer
Данный проект создан исключительно в научно-образовательных целях для изучения реверс-инжиниринга, теории поиска путей (Pathfinding) и симуляции физики 2D-движков. Использование бота на серверах Geometry Dash может привести к бану вашего аккаунта.

---

## 🛠 Архитектура системы

Бот работает по принципу **Frame-Advance Solver (Покадровый математический симулятор)**, а не через анализ картинок (Computer Vision). Это единственный способ добиться точности в 1 кадр на частоте от 240 до 1000+ FPS.

Система разделена на три основных модуля:
1. **GD-Hook-Core (C++)**: DLL-инжектор. Внедряется в процесс игры, перехватывает игровой цикл, замедляет игровое время (Speedhack) и считывает/записывает данные из оперативной памяти.
2. **Solver-Engine (Python / C++)**: Математическое ядро. Считывает координаты хитбоксов и строит дерево возможных путей (Pathfinding) с помощью алгоритмов поиска в ширину (BFS) или модифицированного A*.
3. **Macro-Player (C++)**: Эмулятор ввода. Принимает готовую цепочку успешных кадров и воспроизводит клики через WinAPI (`SendInput`) со стопроцентной точностью на нормальной скорости игры.

## 📋 Техническое задание для AI-Агента (Prompting Guide)

*Инструкция для ИИ-разработчика: Реализуй проект строго по шагам, описанным ниже. Не пиши всё в одном файле.*

### Шаг 1: Чтение памяти и Поиск Офсетов (C++ / Python)
*   **Задача**: Найти базовый адрес игрока (`PlayLayer`), его глобальные координаты `X`, `Y`, текущую скорость `y_velocity` и статус смерти (`isDead`).
*   **Стек**: C++ (`Windows.h`, `TlHelp32.h`) или Python (`pymem`).
*   **Инструкция ИИ**: Создай класс `GDMemoryReader`, который безопасно подключается к `GeometryDash.exe` и выводит текущие координаты куба в консоль в реальном времени.

### Шаг 2: Модификация DeltaTime / Speedhack (C++)
*   **Задача**: Перехватить функцию обновления игры (`update`), чтобы искусственно замедлить процесс (например, до 1 FPS реального времени), давая солверу время «подумать» на каждом кадре.
*   **Стек**: C++ (библиотека MinHook для создания хуков).
*   **Инструкция ИИ**: Напиши код DLL, который подменяет оригинальный `DeltaTime` движка Cocos2d-x на фиксированное минимальное значение.

### Шаг 3: Алгоритм поиска пути (Pathfinding Algorithm)
*   **Задача**: Реализовать алгоритм, который на каждом кадре проверяет два состояния: `Кликнуть (Удерживать)` и `Отпустить`. Бот должен заглядывать вперед на $N$ кадров.
*   **Логика**:
    *   Если при симуляции пути "Отпустить" переменная `isDead == true`, алгоритм откатывается назад и выбирает ветку "Кликнуть".
    *   Для ILL уровней необходимо учитывать хитбоксы. Используй Geode SDK для экспорта структуры объектов текущего уровня.
*   **Инструкция ИИ**: Напиши функцию `predict_next_frames(state, actions)`, реализующую поиск в ширину (BFS) для бинарного дерева решений (0 — отпустить, 1 — нажать).

### Шаг 4: Экспорт макроса
*   **Задача**: При достижении конца уровня (или контрольной точки) сохранить массив пар `(номер_кадра, действие)` в формате JSON или `.gmd`.

## 🚀 Как запустить и протестировать прототип

### Требования
*   Windows 10/11 x64
*   Geometry Dash (Лицензия Steam, версия 2.2+)
*   Python 3.10+ (для математического ядра)
*   Visual Studio 2022 (для сборки C++ DLL)

### Быстрый старт (Разработка)

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com
   cd GD-FramePerfect-Solver
   ```
2. Установите зависимости Python для тестирования логики:
   ```bash
   pip install -r requirements.txt
   ```
3. Откройте решение `GD_Hook.sln` в Visual Studio и скомпилируйте DLL в режиме **Release x86/x64** (в зависимости от разрядности вашей GD).
